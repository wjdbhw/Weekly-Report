# 本周学习周报：Vue快速入门学习与实践总结
## 一、本周学习目标
掌握Vue的核心基础语法与“数据驱动视图”的开发思想，能够独立完成简单Vue项目的初始化、数据绑定及基础交互功能开发。


## 二、核心学习内容与实践
### （一）Vue基础环境与项目初始化
1. **环境搭建方式**
   本次学习采用“直接引入Vue.js文件”的方式（适合入门阶段）：
   - 下载Vue.js文件并存放至项目`js`目录；
   - 在HTML页面的`<head>`或`<body>`底部通过`<script src="js/vue.js"></script>`引入，确保Vue库在业务代码前加载。
   *补充：后续可学习Vue CLI脚手架创建工程化项目，但入门阶段先通过CDN/本地文件快速上手。*

2. **Vue实例的核心配置项**
   通过`new Vue()`创建实例时，重点掌握了2个核心配置：
   - `el: "#app"`：指定Vue实例的“挂载点”（即Vue会管理`id="app"`的DOM节点及内部内容）；
   - `data: { ... }`：定义数据模型（页面中需要渲染/交互的变量都在这里声明，如`message: "Hello Vue!"`）。


### （二）核心语法与视图交互
1. **插值表达式`{{ }}`的应用场景**
   实践了插值表达式的多种用法：
   - 渲染变量：`{{ message }}`直接显示`data`中的`message`值；
   - 算术运算：`{{ 10 + 20 }}`页面显示`30`；
   - 三元运算：`{{ isShow ? "内容可见" : "内容隐藏" }}`（需在`data`中声明`isShow: true`）；
   *注意：插值表达式不能直接写在HTML标签的属性中（如`<div title="{{ message }}">`不生效，需用`v-bind`指令）。*

2. **`v-model`双向绑定的实践**
   完成了“输入框与数据同步”的案例：
   - 在输入框中添加`v-model="message"`，实现“输入框内容变化 → `data.message`同步更新 → 页面中`{{ message }}`自动刷新”的双向联动；
   - 拓展场景：通过`v-model`绑定复选框、单选框，实现多数据的批量同步。


### （三）学习过程中的问题与解决
1. **问题1：Vue实例挂载失败**
   现象：页面未渲染`data`中的数据，控制台提示“Cannot find element: #app”。
   解决：确认`<script>`标签的位置——将创建Vue实例的代码放在`id="app"`的DOM节点**之后**（或用`Vue.$mount()`手动挂载）。

2. **问题2：`v-model`绑定非表单元素无效**
   现象：给`<div>`加`v-model`后无交互效果。
   解决：明确`v-model`是“表单元素专属指令”，仅支持输入框、单选框、下拉框等表单组件，普通DOM元素需用`v-bind`（单向）+ 事件监听（双向）实现交互。


## 三、学习收获与后续计划
1. **本周收获**
   - 建立了“数据驱动视图”的开发思维：不再手动操作DOM，而是通过修改`data`实现页面更新；
   - 掌握了Vue入门的核心流程：“引入Vue → 创建实例 → 配置el/data → 用语法绑定视图”。

2. **后续学习计划**
   - 学习`v-bind`（属性绑定）、`v-on`（事件绑定）等基础指令；
   - 实践简单的交互功能（如按钮点击修改数据、列表渲染）；
   - 了解Vue的生命周期函数，理解实例从创建到销毁的流程。
